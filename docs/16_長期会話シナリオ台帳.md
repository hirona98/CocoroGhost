## セッション共通の5ターン型

1. T1（想起）: 前回までの維持確認質問
2. T2（投入）: 追加または訂正を1件投入
3. T3（確認）: T2が反映されたか確認質問
4. T4（感情）: 感情イベントを1件投入
5. T5（混合）: 維持項目 + 更新項目 + 感情文脈を同時確認

## 期待値の判定ルール

- 訂正が入った項目は、旧情報を出した時点で失点。
- T3は「投入直後の反映確認」。worker同期後に評価する。
- T5は「短期文脈」より「台帳上の真実」を優先しているかを評価する。
- 感情は文言一致ではなく、`mood/debug` の `vad` / `shock_vad` 変化で評価する。

## 実行後に更新する項目（評価記録）

  - 感情反応: OK/NG

- NG時メモ:
  - どのターンで何を誤ったか
  - 関連する `event_id` / `state_id`
  - `worker-stats` と `mood/debug` のスナップショット
