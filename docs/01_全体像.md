## コンポーネント

- APIサーバ
  - `/api/chat`（SSE）
  - 画像/通知/リマインダー等の入口
- 設定DB（SQLite: `settings.db`）
  - トークン / 各種プリセット / 有効フラグ
- リマインダーDB（SQLite: `reminders.db`）
  - リマインダー状態と発火管理
- 記憶DB（SQLite: `memory_<embedding_preset_id>.db`）
  - 出来事ログ（`events`）
  - 状態（`state`）
  - World Model（`wm_*`）
  - Capability Registry（`wm_capabilities` / `wm_capability_operations`）
  - 改訂履歴（`revisions`）
  - 検索インデックス（ベクトル/文字n-gram）
  - 観測ログ（`retrieval_runs`）
  - 感情（`long_mood_state` / `event_affects`）
- 自律基盤（階層型世界モデル）
  - 5ループ（Observe/Strategize/Tacticalize/Execute/Reflect）
  - capability adapter 実行（`capability_id` + `operation`）
  - 行動結果を `events` と `wm_*` へ循環
- LLM（LiteLLM経由）
  - 埋め込み
  - 検索候補選別（`SearchResultPack`）
  - 会話応答（SSE）
  - 記憶更新計画（非同期）
- ルールベース
  - 検索計画（RetrievalPlan）

## `/api/chat` の同期フロー（現行）

体感速度のため SSE は維持し、ストリーム開始前に必要記憶を確定する。

1. `events` を仮作成して `event_id` を確保
2. RetrievalPlan 生成（ルール）
3. 候補収集（ベクトル/文字n-gram/文脈/時間）
4. LLM 選別で `SearchResultPack` 作成
5. LLM 返答を SSE で生成
6. 非同期で記憶更新ジョブを投入

## 非同期フロー（現行）

- `about_time` 推定
- エンティティ抽出
- 状態更新（fact/relation/task/summary）
- `revisions` 追記
- 検索インデックス更新
- 文脈グラフ更新
- 感情更新

## 自律フロー（仕様）

`/api/chat` 同期経路とは分離して worker/periodic 側で実行する。

1. トリガ投入（`event_created` / `action_result` / `periodic` / `startup`）
2. `run_autonomy_cycle` で 5ループ実行
3. command event は `event_id=0` で配信（非永続）
4. result event は `events` へ永続化
5. result event から WritePlan ジョブ投入
6. world model を更新して次サイクルへ循環

## ゴール指標（人間模擬）

1. 長期目標継続性
2. 発話と行動の整合性
3. 失敗後の再計画速度
4. 更新理由の追跡可能性

## コード構成（主要モジュール）

- `cocoro_ghost/memory/`: `/api/chat` 同期フロー
- `cocoro_ghost/worker.py`: 非同期ジョブ実行
- `cocoro_ghost/autonomy/`: 自律ループ/World Model/Registry
- `cocoro_ghost/worker_handlers_autonomy.py`: 自律ジョブハンドラ
- `cocoro_ghost/prompt_builders.py`: プロンプト組み立て
- `cocoro_ghost/affect.py`: 感情更新ロジック
- `cocoro_ghost/vector_index.py`: ベクトル索引
