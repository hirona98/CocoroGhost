# 階層型世界モデル 実装計画

## 1. 目的

本計画は、階層型世界モデルの導入を
「概要 -> 仕様 -> 実装」の順で段階的に進めるための実装計画である。

この計画書は、着手前に合意し、以後の作業の基準とする。

## 2. 前提

1. capability 一覧は固定しない
2. 最初の具体能力は `web_access` だが、専用設計にはしない
3. `web_access` の設計は最後に行う
4. `/api/chat` の同期経路を維持し、重い処理は worker/periodic 側に置く
5. 既存仕様（Web検索フラグ、命令イベント非永続化）と矛盾しないこと

## 3. 実装方針

1. Gate 管理で進める（各Gateでユーザー確認）
2. 1フェーズごとに文書を確定してから次へ進む
3. 設計が固まるまで実装へ進まない
4. 既存ドキュメントは「まとめて更新」ではなく、フェーズ単位で追従更新する

## 7. リスクと対策

1. リスク: 基盤より先に能力詳細へ寄る
   - 対策: `web_access` は Phase 5 まで仕様対象外として扱う
2. リスク: `/api/chat` 経路への処理混入
   - 対策: 実装レビュー時に同期/非同期境界チェックを必須化する
3. リスク: 既存仕様との矛盾
   - 対策: 各Phaseで `docs/10_実行フロー.md` と `docs/07_API.md` を突合する
