# 階層型世界モデル 実装計画

## 1. 目的

本計画は、階層型世界モデル導入を
「概要 -> 仕様確定 -> 実装 -> 検証」の順で進めるための実装計画である。

本計画の最終目的は、人間らしい連続性を持つ AI 人格を成立させること。

## 2. 前提

1. capability 一覧は固定しない
2. 基盤契約（goal/ticket/result/observation/capability）は先に固定する
3. `/api/chat` 同期経路に重い処理を混ぜない
4. fail-fast を正とし、fallback を導入しない
5. `web_access` は capability 共通契約で実装し、専用経路を作らない

## 3. 進め方

1. Gate 管理で進める（Gate承認なしで次フェーズへ進まない）
2. 1フェーズごとに仕様を確定し、相互矛盾を解消してから次へ進む
3. 仕様確定前にコードへ入らない
4. 既存 docs はフェーズ単位で追従更新する
